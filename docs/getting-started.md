# Getting Started

MissionOS boots into a first-run flow:

1. Create the first local account.
2. Create the singleton project.
3. Onboard the first agent.
4. Land on the dashboard and org chart.

The app stores all state in `server/data/missionos.db`.

## Core areas

- Missions coordinate teams and goals.
- Agents define execution endpoints and prompt scaffolding.
- Issues track work in list or board mode.
- Runs record live execution output.
- Settings manage auth profile, Linear, and engines.
